# 6f — Проект в GTD — больше одного действия

> [!abstract] Определение
> У Аллена **проект** — это **желаемый исход**, для достижения которого нужно **больше одного** физического действия. Пока исход не достигнут (или не отменён), проект **открыт** и тянет за собой список **next actions** и напоминания.

## Зачем так резать

Если всё свалить в «одну задачу», теряется **контроль**: непонятно, **какой** следующий шаг сдвинет исход. Разбиение на проект + конкретные next actions — способ держать **горизонтальное** управление без перегруза ОЗУ (см. [[Б37 — Аллен — Getting Things Done]]).

## Стык с Projects в PARA

В [[8 — PARA — сортировка по активности, а не по теме|PARA]] **Projects** — зона **конечных исходов** и активного движения. GTD-определение «> одного шага» хорошо **стыкуется** с кислородом проектов в той же заметке: без живых проектов критерий активности **не к чему приложить**.
