# 8 — PARA — сортировка по активности, а не по теме

> [!abstract] Один критерий
> Четыре корзины **PARA** ([[Б2 — Форте 2024 — Создавая второй мозг|Форте]]) — это **не** «четыре темы», а **одна ось**: **насколько активно** материал **двигает обязательства вперёд**. В духе **[[Б37 — Аллен — Getting Things Done|GTD]]** вопрос звучит жёстко и полезно: **«какой проект это двигает *сейчас*?»**

## Четыре категории (смысл с карточки)

| | **Что это** | **Ключ** |
|---|-------------|----------|
| **Projects** | Цель + **дедлайн** / конечный результат | **Конечно** — здесь «горит» |
| **Areas** | **Бессрочная** ответственность: стандарты, роли, поддержка | Нет финальной даты, но нужно **регулярное** внимание |
| **Resources** | Интересы и темы; **топливо** будущих проектов | «Пригодится», но **не** тянет календарь сегодня |
| **Archives** | **Завершённое** | **Морозильник**, не свалка: можно **разморозить**, если контекст вернулся |

На [[00 Главная|главной]] уже намечены узлы `PARA — …` — это и есть «витрина» четырёх режимов **активности**.

## Тематическая сортировка ломает выбор

> «**Дофамин** — это нейронаука? Продуктивность? Маркетинг?» → **паралич** выбора папки.

**PARA** снимает драму «правильной полки»: вопрос не «**о чём** это?», а «**что это делает для проектов *прямо сейчас***?» — и остаётся **одно** следующее действие без мучений.

## Проекты = кислород

> [!warning] Без проектов PARA «пустая»
> Если **нет** живых **[[PARA — Проекты|проектов]]**, PARA превращается в **косметику**: корзины есть, а **критерий активности** не к чему приложить. **Проекты** — **кислород** системы: они задают, зачем вообще течёт поток в **Areas / Resources**. В терминах GTD проект — исход с **больше чем одним** физическим шагом — см. [[6f — Проект в GTD — больше одного действия]].
