# 6e — Inbox и сбор в GTD

> [!abstract] Различие
> **Collect** в GTD — это **корзины сбора**: всё незавершённое вытащить из головы **быстро**, без преждевременной шлифовки. **Хранить** там вечно нельзя — корзины нужно **регулярно опустошать** на стадии **process** / **organize**. Путаница «инбокс = склад» ломает доверие к системе.

## «Inbox zero» как культура vs механика Аллена

Маркетинговый образ **нуля входящих** иногда смешивают с идеей GTD. У Аллена в центре не красота пустого ящика, а **ясность**: каждый вход обработан решением (мусор / инкубация / справочник / действие с next step). Нулевой инбокс — **следствие** дисциплины обработки, а не самоцель.

## Стык с PARA и CODE

В **[[8 — PARA — сортировка по активности, а не по теме|PARA]]** вопрос после обработки: **насколько активно** материал двигает обязательства **сейчас** — тема папки вторична. У Форте стадия **Capture** в [[Б2 — Форте 2024 — Создавая второй мозг|CODE]] близка по духу: **сначала** поймать сырьё, **потом** раскладывать под действие. Разница в акцентах раскрыта в [[6a — PARA как мост GTD–PKM]].
